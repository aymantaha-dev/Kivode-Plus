# Submission Process

## Overview

Developer Submissions allow maintainers of Community Builds to request Directory Listing on the official Kivode platform.

Submission URL:

https://kivode.com/apps/submit

## Process Stages

1. **Preparation**
   - Confirm your repository is public and accessible.
   - Confirm your build process is documented and reproducible.
   - Prepare screenshots and a clear project description.

2. **Submission**
   - Submit repository details and supporting content through the submission form.
   - Include developer profile and contact links.

3. **Review**
   - The Kivode maintainers review technical quality, safety, and license compliance.
   - Additional clarification may be requested.

4. **Decision**
   - Approved projects are eligible for Directory Listing.
   - Rejected projects may be resubmitted after issues are resolved.

## What Maintainers Evaluate

- Build integrity and documentation quality.
- Security posture and safe execution behavior.
- Compliance with non-commercial license terms.
- Repository transparency and attribution.
