# Directory Model

## Purpose

Kivode+ is governed as a **Reference Implementation** with an open ecosystem of **Community Builds**.

The official repository provides a stable baseline implementation. Community developers are encouraged to create independent versions in their own repositories.

## Core Principles

- The official repository is the Reference Implementation.
- Community innovation happens in Independent Fork repositories.
- Community projects remain the responsibility of their own maintainers.
- The official project may review and include selected projects in a public Directory Listing.

## Why This Model Exists

This model allows:

- strong governance and stability in the official codebase,
- rapid experimentation by community maintainers,
- transparent separation of ownership,
- a curated directory that helps users discover high-quality variants.

## How Community Builds Fit In

A Community Build is any independent project derived from Kivode+ and hosted in a separate repository.

Community maintainers may:

- modify architecture and product behavior,
- add or remove features,
- rename and brand their version,
- publish releases under their own project identity,

as long as they respect license terms and non-commercial restrictions.

## Directory Listing

Projects can be submitted through the Developer Submissions portal:

https://kivode.com/apps/submit

Approved listings may include project information such as name, description, repository link, screenshots, and developer profile links.
