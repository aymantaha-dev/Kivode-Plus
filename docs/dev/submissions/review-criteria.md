# Review Criteria

## Reasons a Project May Be Rejected

A project may be rejected for reasons including:

- security vulnerabilities,
- unsafe preview execution,
- exposed API keys or secrets,
- malicious code,
- commercial distribution violating the license,
- broken build process,
- plagiarism without attribution.

## What Makes a Project More Likely to Be Accepted

A project is more likely to be accepted when it demonstrates:

- clear documentation,
- working build process,
- useful improvements or features,
- stable code quality,
- clean repository structure,
- active maintenance,
- screenshots and description.
