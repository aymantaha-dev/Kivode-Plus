# Developer Submissions Guidelines

## How to Submit Your Project

1. Create your repository.
2. Ensure the project builds successfully.
3. Prepare screenshots.
4. Write a clear project description.
5. Submit the project through:

https://kivode.com/apps/submit

## Required Submission Content

Each submission must include:

- repository link,
- project description,
- screenshots,
- developer contact links.

## Recommended Quality Signals

To improve review speed and quality, include:

- a complete README with setup instructions,
- explicit license and attribution information,
- release notes or change summary compared to the Reference Implementation,
- known limitations and platform support details.

## After Submission

- The project enters technical and policy review.
- Maintainers may request clarifications or fixes.
- Approved projects become eligible for official Directory Listing.
